# Настройка «Сохранить для всех»

Этот воркер — единственная часть системы, которая умеет писать в GitHub. Он живёт
отдельно от сайта (сайт на GitHub Pages статичный и ничего писать не может), крутится
на бесплатном тарифе Cloudflare Workers и коммитит `prices.json` в репозиторий по запросу
от кнопки «Сохранить для всех».

Всё ниже делаешь только ты, через свои личные аккаунты. Я не могу создать токен или
аккаунт за тебя, и токен GitHub никогда не должен попадать в чат/код/куда-либо публично —
он живёт только в секретах Cloudflare.

## Шаг 1. GitHub-токен (Personal Access Token)

1. GitHub → своя аватарка (правый верхний угол) → **Settings** → в самом низу слева
   **Developer settings** → **Personal access tokens** → **Fine-grained tokens** →
   **Generate new token**.
2. **Repository access**: `Only select repositories` → выбрать
   `GeeFestTech/-RSCP-Polymer-Calculator` (именно этот один репозиторий, не «All repositories»).
3. **Permissions** → **Repository permissions** → `Contents` → **Read and write**.
   Остальные права оставить `No access`.
4. Срок действия — можно ограничить (например, 1 год), потом просто перевыпустишь.
5. Сгенерировать, скопировать токен (он покажется один раз) — сразу вставишь в Cloudflare
   на шаге 3, никуда больше не сохраняй его текстом.

## Шаг 2. Cloudflare-аккаунт и Worker

1. Зарегистрируйся (бесплатно) на [dash.cloudflare.com](https://dash.cloudflare.com) —
   если аккаунта ещё нет.
2. В боковом меню **Workers & Pages** → **Create** → **Create Worker**.
3. Дай имя, например `eve-polymer-price-sync` (можно любое — это станет частью URL:
   `https://<имя>.<твой-поддомен>.workers.dev`).
4. После создания открой воркер → **Edit code** (Quick Edit) и полностью замени
   содержимое на код из [worker.js](worker.js) из этой папки.
5. **Deploy**.

## Шаг 3. Секреты воркера

В настройках воркера: **Settings** → **Variables and Secrets** → **Add**:

| Имя | Тип | Значение |
|---|---|---|
| `GITHUB_TOKEN` | Secret | токен из Шага 1 |
| `SAVE_PASSWORD` | Secret | пароль, который сам придумаешь — его будет спрашивать кнопка на сайте |

Оба поля отмечай как **Secret** (не Text) — тогда значение не будет видно даже тебе
после сохранения, только воркеру во время выполнения.

## Шаг 4. Впиши адрес воркера в сайт

1. Скопируй финальный URL воркера (Cloudflare покажет его на странице воркера,
   вида `https://eve-polymer-price-sync.твой-поддомен.workers.dev`).
2. В [index.html](../index.html) найди строку:
   ```js
   const SAVE_ENDPOINT = "https://REPLACE-WITH-YOUR-WORKER-URL.workers.dev/save";
   ```
   и замени на свой URL с припиской `/save` в конце, например:
   ```js
   const SAVE_ENDPOINT = "https://eve-polymer-price-sync.твой-поддомен.workers.dev/save";
   ```
3. Загрузи обновлённый `index.html` в репозиторий тем же способом, каким обычно
   деплоишь (Upload files на github.com).

## Шаг 5. Проверка

1. Открой сайт, зайди на вкладку «Рынок цен», поменяй любую цену.
2. Нажми «🌍 Сохранить для всех», подтверди, введи пароль из Шага 3.
3. Должно появиться сообщение «Готово!» со ссылкой на коммит в GitHub — открой её,
   убедись, что `prices.json` в репозитории обновился.
4. Открой сайт в приватной вкладке браузера (без своего localStorage) — новые цены
   должны подтянуться как заводские.

## Если что-то не так

- **«Неверный пароль»** — проверь `SAVE_PASSWORD` в секретах воркера и что вводишь
  то же самое на сайте.
- **CORS-ошибка в консоли браузера** — проверь константу `ALLOWED_ORIGIN` в
  `worker.js`, она должна точно совпадать с адресом твоего GitHub Pages
  (сейчас `https://geefesttech.github.io`).
- **«GitHub отклонил запись»** — скорее всего токен просрочен, отозван или у него нет
  права `Contents: Read and write` именно на этот репозиторий. Перевыпусти токен
  (Шаг 1) и обнови секрет `GITHUB_TOKEN` в Cloudflare.
